# Code of Conduct

## Our standard

This project welcomes contributions from anyone willing to engage with it constructively. We expect everyone interacting in this project — issues, pull requests, discussions, security advisories — to act in good faith.

In practice that means:

- **Stay technical.** Critique code, designs, and citations; not the people who wrote them.
- **Assume good intent.** A first-time contributor's misunderstanding deserves a kind clarification, not a sarcastic reply.
- **Be concrete.** "This is wrong" carries less weight than "this is wrong because X; here's the source." The whole skill is built on this discipline.
- **Cite when you correct.** Especially for the citation-error workflow — your verification source is what makes the correction trustworthy.
- **Respect the bar.** The skill is curated, not assembled-by-committee. Proposals that don't meet the documented standards may be politely deferred or declined.

## Not in scope

- Personal attacks of any kind.
- Off-topic political or ideological arguments.
- Spam, automated mass-generated content, low-effort issue submissions.
- Disclosing private security vulnerabilities in public issues (use the [Security policy](.github/SECURITY.md) instead).

## Enforcement

This is a solo-maintainer project. The maintainer reserves the right to:

- Delete comments that violate this code.
- Close issues or PRs without further discussion when the interaction has gone off-track.
- Block users who repeatedly violate the standard.

Enforcement decisions are documented when meaningful (e.g. in a closed issue's comments). Disagreements with an enforcement decision can be raised by opening a calm, specific issue or by emailing the maintainer.

## Reporting concerns

To report a concern about another participant's conduct in this project:

- Open a [GitHub Security Advisory](https://github.com/franciscolopes26/staff-engineer-mode/security/advisories/new) — these are private to the maintainer.
- Or email the maintainer (contact via the GitHub profile of [@franciscolopes26](https://github.com/franciscolopes26)).

Reports are handled with discretion. The reporter's identity is not shared with the reported party without consent.

## Scope

This code applies in all project spaces — the GitHub repository (issues, PRs, discussions, security advisories), and any related project communication channels that exist now or in the future.

## Inspiration

This code is intentionally short. For projects that want the longer, more formal alternative, see the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
